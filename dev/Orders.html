This is specialized by MonomialOrder in case of Buchberger's algorithm and by ModuleMonomialOrder in case of Signature-based algorithms.

IPGBs.Orders.MonomialOrder — Type

Implements a monomial order from a given cost matrix, including a grevlex tiebreaker if necessary.

The cost matrix is represented in column-major order to speed up comparisons.

IPGBs.Orders.is_inverted_invlex — Method
is_inverted_invlex(order :: MonomialOrder, v :: T) :: Bool where {T <: AbstractVector{Int}}

Return true iff the trailing and leading terms of v are inverted with respect to the invlex order.

v is inverted according to invlex iff its first non-zero entry is positive. Inversion testing is thus done more efficiently in the invlex case.
